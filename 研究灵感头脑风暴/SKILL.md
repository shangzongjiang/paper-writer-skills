# 研究创意头脑风暴

用于发现下一个研究方向的结构化框架。本技能提供十种互补的创意思考视角，帮助研究者从模糊的好奇心出发，形成具体且有说服力的研究提案。每种框架针对不同的认知模式——可单独使用，也可组合运用以实现全面探索。

## 适用场景

- 开启新研究方向，需要结构化探索
- 在当前项目上遭遇瓶颈，希望获得新视角
- 评估一个半成形想法是否具有真正潜力
- 为与合作者的头脑风暴会议做准备
- 在研究领域间转换，寻找高杠杆的切入点
- 回顾某一领域，寻找尚未充分探索的缺口

**以下情况请勿使用本技能**：
- 你已有明确的研究问题，需要的是执行指导
- 需要实验设计或方法论帮助（请使用领域专属技能）
- 需要文献综述（请使用 `scientific-skills:literature-review`）

## 核心创意框架

### 1. 问题优先 vs 方案优先思维

研究创意来源于两种截然不同的模式。明确自己处于哪种模式，可以避免一个常见失败：构建了没有真实问题的解决方案，或者追逐没有可行方法的问题。

**问题优先**（痛点 → 方法）：
- 从具体的失败、瓶颈或未满足的需求出发
- 由于动机内在，自然产生有影响力的工作
- 风险：可能趋向渐进式修补，而非范式转变

**方案优先**（新能力 → 应用）：
- 从寻求应用场景的新工具、洞见或技术出发
- 通过解锁之前不可能的方法，往往推动突破性进展
- 风险："手握锤子找钉子"——解决方案可能缺乏真实需求

**工作流**：
1. 用一句话写下你的想法
2. 分类：这是问题优先还是方案优先？
3. 若是问题优先 → 验证问题的重要性（谁受困？程度如何？）
4. 若是方案优先 → 识别出它所解决的至少两个真实问题
5. 无论哪种模式，都要明确差距：这能实现哪些今天无法做到的事？

**自查清单**：
- [ ] 我能说出一个具体的、需要这项成果的人或社群吗？
- [ ] 我要解决的问题真的没有被解决（而不仅仅是营销不足）？
- [ ] 若是方案优先，该方案是否创造了新能力，还是只是复制了已有能力？

### 2. 抽象阶梯

每个研究问题都处于特定的抽象层次。有意识地上下移动阶梯，可以发现在当前层次看不到的创意。

| 方向 | 操作 | 结果 |
|------|------|------|
| **向上**（泛化） | 将具体结果提升为更宏观的原理 | 框架类论文、理论贡献 |
| **向下**（实例化） | 在具体约束下检验通用范式 | 实证类论文、令人惊讶的失败分析 |
| **横向**（类比） | 将相同抽象层次应用于相邻领域 | 跨学科迁移、知识转移类论文 |

**工作流**：
1. 用一句话陈述你当前的研究焦点
2. 向上移动：背后的普遍原理是什么？这属于哪类问题？
3. 向下移动：这个问题最具体、最受约束的实例是什么？极端情况下会发生什么？
4. 横向移动：这个模式在其他领域中以何种形式出现？
5. 对每个新层次，问：这本身是否构成一个可发表的贡献？

**示例**：
- **当前**："提高 RAG 系统的检索准确率"
- **向上**："什么使上下文选择对任何增强生成系统有效？"
- **向下**："当文档被对抗性扰动时，检索准确率如何下降？"
- **横向**："数据库查询优化使用了类似的相关性排序——我们能借鉴什么？"

### 3. 张力与矛盾猎寻

突破往往来自化解广泛认可却看似矛盾的目标之间的张力。这些矛盾不是缺陷——而是研究机会。

**常见研究张力**：

| 张力对 | 研究机会 |
|--------|----------|
| 性能 ↔ 效率 | 能否以 10 倍更少的算力达到 SOTA？ |
| 隐私 ↔ 效用 | 联邦/加密方法能否缩小准确率差距？ |
| 通用性 ↔ 专业化 | 何时微调优于提示，为什么？ |
| 安全 ↔ 能力 | 对齐能否提升而非削弱能力？ |
| 可解释性 ↔ 性能 | 机制洞察能否带来更好的架构？ |
| 规模 ↔ 可及性 | 小模型能否复现涌现行为？ |

**工作流**：
1. 选定你的研究领域
2. 列出 3-5 个最受重视的目标
3. 识别通常被视为权衡的目标对
4. 对每对，问：这种权衡是根本性的，还是当前方法的产物？
5. 若是方法产物 → 化解它本身就是你的研究贡献
6. 若是根本性的 → 刻画 Pareto 前沿本身也有价值

**自查清单**：
- [ ] 我确认这种张力是真实存在的（而非假设）？
- [ ] 我能指出各自优化一侧的论文吗？
- [ ] 我提出的化解方案在技术上可行，而不仅仅是愿景？

### 4. 跨学科迁移（类比转换）

从其他学科借鉴结构性思路是最富生产力的研究启发之一。许多基础技术正是这样诞生的——注意力机制来自认知科学，遗传算法来自生物学，对抗训练来自博弈论。

**有效类比的要求**：
- **结构保真度**：映射必须在底层机制层面成立，而非仅停留于表面相似
- **非显见联系**：若该联系已广为人知，则新颖性已不再
- **可检验预测**：类比应能产生具体的假设

**机器学习研究的高产出源头领域**：

| 源头领域 | 可迁移概念 |
|----------|------------|
| 神经科学 | 注意力机制、记忆巩固、分层处理 |
| 物理学 | 基于能量的模型、相变、重整化 |
| 经济学 | 机制设计、拍卖理论、激励对齐 |
| 生态学 | 种群动力学、生态位竞争、协同演化 |
| 语言学 | 组合性、语用学、语法归纳 |
| 控制论 | 反馈回路、稳定性、自适应调节 |

**工作流**：
1. 用去除专业术语的领域无关语言描述你的问题
2. 问：哪个领域解决了结构上相似的问题？
3. 在机制层面研究该领域的解决方案
4. 将解决方案映射回你的领域，保留结构关系
5. 从类比中生成可检验的预测
6. 验证：借鉴的思路是否真正改善了结果？

### 5. "什么改变了？"原则

强有力的创意往往来自在新条件下重审旧问题。硬件、规模、数据可用性或法规的进步，可能使之前的假设失效，让原本不切实际的方法变得可行。

**需要关注的变化类别**：

| 变化类型 | 示例 | 研究意涵 |
|----------|------|----------|
| **算力** | GPU 性能提升 10 倍 | 曾被认为过于昂贵的方法变得可行 |
| **规模** | 万亿 token 数据集 | 小规模失败的统计论证现在可能成立 |
| **法规** | 欧盟 AI 法案、GDPR | 产生对合规替代方案的需求 |
| **工具链** | 新框架、新 API | 降低复杂方法的实现门槛 |
| **失败案例** | 高关注度系统故障 | 暴露现有方法的缺口 |
| **文化** | 用户行为变化 | 改变最重要的问题 |

**工作流**：
1. 选取一个知名的负面结论或已被放弃的方法（3-10 年前）
2. 列出导致其被否定的假设条件
3. 对每个假设，问：这在今天仍然成立吗？
4. 若任何假设已被推翻 → 在新条件下重新检验这个想法
5. 框定贡献："X 以前因 Y 不可行，但 Z 已经改变"

### 6. 失败分析与边界探测

理解一种方法在哪里失效，往往与展示它在哪里有效同等重要。边界探测系统性地揭示公认技术失效的条件。

**待探测的边界类型**：
- **分布性**：分布外输入时会发生什么？
- **规模性**：方法在 10 倍或 0.1 倍典型规模下会退化吗？
- **对抗性**：方法能被有意破坏吗？
- **组合性**：结合多种能力时，性能是否保持？
- **时间性**：方法随时间退化吗（概念漂移）？

**工作流**：
1. 选择一个有强烈已报告结果的广泛使用方法
2. 识别其评估中隐含的假设（数据集、规模、领域）
3. 系统性地违反每个假设
4. 记录方法在何处及如何失效
5. 诊断每次失败的根本原因
6. 提出修复方案，或解释为何该失败是根本性的

**自查清单**：
- [ ] 我在探测真正的边界，而不是仅仅确认已知局限？
- [ ] 我能解释方法*为何*失败，而不只是*它*失败？
- [ ] 我的分析是否指向了一条建设性的前进路径？

### 7. 简洁性测试

在接受复杂度之前，先问简单方法是否足够。领域有时过度依赖精密解决方案，而简化的基线在竞争性上毫不逊色。

**不必要复杂度的预警信号**：
- 方法有许多窄最优范围的超参数
- 消融研究显示大多数组件贡献边际
- 一个简单基线从未被适当调优或评估
- 在大多数基准上相对基线的提升在噪声范围内

**工作流**：
1. 识别你所在问题的当前 SOTA 方法
2. 将其剥离至最简核心（唯一的关键思路是什么？）
3. 用精心的工程实现最小化版本
4. 公平比较：相同算力预算，相同调优努力
5. 若差距小 → 贡献本身就是简洁性
6. 若差距大 → 你现在理解了复杂度换来的是什么

**贡献表述框架**：
- "我们展示了带有[一个改动]的[简单方法]与[复杂 SOTA]相当"
- "我们识别出[特定组件]是关键驱动力，而非[其他组件]"

### 8. 利益相关者轮换

从多个视角审视一个系统，揭示不同类别的研究问题。每个利益相关者看到不同的摩擦、风险和机会。

**利益相关者视角**：

| 利益相关者 | 关键问题 |
|------------|----------|
| **终端用户** | 这可用吗？哪些错误不可接受？延迟容忍度是多少？ |
| **开发者** | 这可调试吗？维护负担如何？如何组合使用？ |
| **理论家** | 这为什么有效？形式保证是什么？缺口在哪里？ |
| **对手** | 如何利用这个系统？攻击面在哪里？ |
| **伦理学家** | 谁受到伤害？嵌入了哪些偏见？谁被排除在外？ |
| **监管者** | 这可被审计吗？决策能被解释吗？是否有问责机制？ |
| **运营者** | 成本是多少？如何扩展？失败模式是什么？ |

**工作流**：
1. 用一段话描述你的系统或方法
2. 依次假设每个利益相关者视角（每个角色花 5 分钟）
3. 对每个视角，列出前 3 个关切或问题
4. 识别现有工作未解决的关切
5. 影响范围最广的未解决关切就是你的研究问题

---

### 9. 组合与分解

新颖性往往来自重组或模块化。创新往往不在于新的基本原语，而在于组件的排列或拆分方式。

**组合**（结合现有技术）：
- 识别解决互补子问题的两种方法
- 问：组合它们会产生什么涌现能力？
- 示例：RAG + Chain-of-Thought → 检索增强推理

**分解**（拆解单体系统）：
- 识别一个组件相互纠缠的复杂系统
- 问：哪个组件是真正的瓶颈？
- 示例：将"微调"分解为数据选择、优化和正则化，发现数据选择通常最重要

**工作流**：
1. 列出你所在领域的 5-10 个核心组件或技术
2. **组合**：选择配对，问组合后会发生什么
3. **分解**：选取一个复杂方法，分离每个组件的贡献
4. 对于组合：组合是否产生了涌现能力？
5. 对于分解：分离是否揭示了主导或冗余组件？

### 10. "向他人解释"测试

一个强研究创意应该能向聪明的非专家用两句话进行辩护。这个测试强制要求目的清晰，磨砺价值主张。

**两句话模板**：
> **第一句**（问题）："[领域] 目前面临 [具体问题] 的困难，这很重要因为 [具体后果]。"
> **第二句**（洞见）："我们通过 [关键机制] 来 [方法]，这有效因为 [原因]。"

**若你无法填完此模板**：
- 问题可能尚未明确 → 回到框架 1
- 洞见可能尚不清晰 → 回到框架 7（简化）
- 重要性可能尚未确立 → 回到框架 3（找到张力）

**校准问题**：
- 你所在子领域以外的聪明同事能理解为什么这很重要吗？
- 解释能否在不用术语的情况下成立？
- 你能预测怀疑者最先提出的反对意见吗？

## 综合头脑风暴工作流

使用这个端到端工作流，从白纸到排名好的研究创意。

### 阶段一：发散（生成候选）

**目标**：不加筛选地产生 10-20 个候选创意。

1. **扫描张力**（框架 3）：列出你所在领域的 5 个权衡
2. **检查变化**（框架 5）：列出 3 个近期转变（算力、数据、法规）
3. **探测边界**（框架 6）：选 2 个流行方法，找到它们失效的地方
4. **跨学科迁移**（框架 4）：从相邻领域选 1 个想法
5. **组合/分解**（框架 9）：组合 2 种现有技术，或拆解 1 种技术
6. **爬抽象阶梯**（框架 2）：对每个候选，生成上/下/横向变体

### 阶段二：收敛（筛选与排名）

**目标**：缩窄至 3-5 个最强创意。

对每个候选应用以下过滤器：

| 过滤器 | 问题 | 淘汰标准 |
|--------|------|----------|
| **解释测试**（F10） | 能用两句话陈述吗？ | 若不能 → 创意尚不清晰 |
| **问题优先检查**（F1） | 问题是否真实且重要？ | 若无人受困 → 放弃 |
| **简洁性测试**（F7） | 复杂度是否合理？ | 若更简单方法有效 → 简化或放弃 |
| **利益相关者检查**（F8） | 谁受益？谁可能反对？ | 若无明确受益者 → 放弃 |
| **可行性** | 能用现有资源执行吗？ | 若明显不可行 → 搁置待以后 |

### 阶段三：精炼（打磨获胜创意）

**目标**：将最优创意转化为具体研究计划。

1. 撰写两句话简报（框架 10）
2. 识别被化解的核心张力（框架 3）
3. 明确抽象层次（框架 2）
4. 列出 3 个能验证该创意的具体实验
5. 预判最强的反对意见并准备回应
6. 定义一个 2 周的先导实验，提供可行性信号

**完成检查清单**：
- [ ] 两句话简报清晰有力
- [ ] 问题是真实的（问题优先检查通过）
- [ ] 方法是合理的（简洁性测试通过）
- [ ] 至少一个利益相关者明确受益
- [ ] 核心实验已明确
- [ ] 可行性先导实验已定义
- [ ] 最强反对意见已有回应

## 框架选择指南

不确定从哪个框架入手？使用下面的决策指南：

| 你的处境 | 从此开始 |
|----------|----------|
| "我不知道该研究哪个领域" | 张力猎寻（F3）→ 什么改变了（F5） |
| "我有模糊方向但无具体想法" | 抽象阶梯（F2）→ 失败分析（F6） |
| "我有想法但不确定是否够好" | 解释测试（F10）→ 简洁性测试（F7） |
| "我有好想法但需要新角度" | 跨学科迁移（F4）→ 利益相关者轮换（F8） |
| "我想把现有工作组合成新东西" | 组合/分解（F9） |
| "我发现了酷技术想应用它" | 问题优先检查（F1）→ 利益相关者轮换（F8） |
| "我想挑战传统观念" | 失败分析（F6）→ 简洁性测试（F7） |

## 研究创意构思的常见陷阱

| 陷阱 | 症状 | 修复 |
|------|------|------|
| **新颖但无影响** | "没人做过 X"，但也没人需要 X | 应用问题优先检查（F1） |
| **默认渐进** | 想法只是在某个基准上 +2% | 爬抽象阶梯（F2） |
| **崇拜复杂度** | 方法有 8 个组件，每个贡献微乎其微 | 应用简洁性测试（F7） |
| **回音室效应** | 所有想法来自读同样的 10 篇论文 | 使用跨学科迁移（F4） |
| **陈旧假设** | "这个被试过，不管用"（5 年前） | 应用什么改变了（F5） |
| **单一视角偏差** | 只考虑机器学习工程师的视角 | 使用利益相关者轮换（F8） |
| **过早收敛** | 未探索替代方案就认定了第一个想法 | 完整运行发散阶段 |

## 智能体使用说明

当研究者寻求研究创意头脑风暴帮助时：

1. **确定起点**：他们是在探索新领域、困于当前项目，还是在评估已有想法？
2. **选择合适框架**：使用框架选择指南，挑选 2-3 个相关视角
3. **互动式走过框架**：逐步应用每个框架，请研究者提供领域特定输入
4. **生成候选**：在各框架中争取产出 10-20 个原始想法
5. **筛选与排名**：应用收敛阶段过滤器，缩窄至前 3-5 个
6. **精炼获胜者**：帮助阐明两句话简报，定义具体的后续步骤

**核心原则**：
- 推动具体性——模糊想法（"提高效率"）无法付诸行动
- 挑战假设——至少追问三次"为什么"
- 维护所有候选的书面记录，即使被拒绝的（它们以后可能重新组合）
- 研究者最终决定追求哪些想法；智能体负责促进结构化思考
