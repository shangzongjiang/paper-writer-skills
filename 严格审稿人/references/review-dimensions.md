# 二级审查维度 — 评分锚点与检查清单

六个认识论质量维度。所有检查均为语义层面：需要对 ARA 内容进行阅读理解和推理。结构验证（引用解析、字段存在性、YAML 解析）完全由一级处理。

---

## D1. 证据相关性

**问题**：所引用的证据是否在实质内容上（而非仅仅在引用形式上）支持每项声明？

### 检查

| 检查项 | 需核实的内容 | 发现严重程度 |
|-------|---------------|-----------------|
| 相关性 | 实验的 Setup/Procedure 是否真正针对声明所断言的内容 | 重大 |
| 类型感知蕴含 | 实验设计是否与声明类型匹配（因果→消融实验，泛化→异质性，改进→基线，描述性→采样，范围→边界） | 重大 |
| 证据充分性 | 单一实验是否足以支持该声明，还是需要多个实验？ | 建议 |

### 评分锚点

| 分数 | 描述 |
|-------|-------------|
| 5 | 每项声明均有类型适当、相关的证据；必要时有多实验支撑 |
| 4 | 所有声明的证据均相关，存在轻微类型不匹配 |
| 3 | 大多数声明-实验对相关，1-2 个匹配薄弱 |
| 2 | 多项声明所引用的实验未能实质性地支持该声明 |
| 1 | 大多数声明引用的实验与其陈述无关 |

---

## D2. 可证伪性质量

**问题**：声明是否具有真正可证伪性，并具备有意义、可操作的判定标准？

### 检查

| 检查项 | 需核实的内容 | 发现严重程度 |
|-------|---------------|-----------------|
| 可操作性 | 独立研究人员能否执行？是否明确了测量内容、失败阈值和条件？ | 重大 |
| 非平凡性 | 判定标准是否超越同义反复？（"如果方法不奏效"= 平凡） | 重大 |
| 范围匹配 | 判定标准是否与声明覆盖相同的范围？ | 重大 |
| 独立性 | 是否可以在不使用专有数据或系统的情况下进行测试？ | 次要 |

### 评分锚点

| 分数 | 描述 |
|-------|-------------|
| 5 | 每项声明均有具体、可操作、可独立检验且与声明范围匹配的判定标准 |
| 4 | 大多数标准较强，1-2 个模糊或难以操作化 |
| 3 | 参差不齐；部分可操作，部分平凡或范围不匹配 |
| 2 | 大多数标准平凡、同义反复或范围不匹配 |
| 1 | 所有声明的判定标准均无实质意义 |

---

## D3. 范围校准

**问题**：声明所断言的内容是否恰好与其证据所支持的范围吻合——不多也不少？

### 检查

| 检查项 | 需核实的内容 | 发现严重程度 |
|-------|---------------|-----------------|
| 过度声明 | 声明使用了全称范围，而证据仅覆盖有限条件 | 极端情况为关键，中等情况为重大 |
| 欠充分声明 | 证据文件或实验结果未被任何声明所涵盖 | 次要 |
| 假设显式性 | 关键假设是否在 problem.md 或 constraints.md 中有所陈述 | 若未陈述的假设影响有效性则为重大 |
| 泛化边界 | Artifact 是否说明了声明不适用的情形 | 次要 |
| 限定词一致性 | 对冲语言是否与证据强度相匹配 | 次要 |

### 评分锚点

| 分数 | 描述 |
|-------|-------------|
| 5 | 所有声明与证据范围精确匹配，假设明确，局限性有所陈述 |
| 4 | 范围校准良好，假设文档存在轻微缺口 |
| 3 | 部分声明略有过度/欠充分，假设部分陈述 |
| 2 | 多处过度声明或存在大量未记录的假设 |
| 1 | 声明与证据之间的范围普遍不匹配 |

---

## D4. 论证连贯性

**问题**：论证是否遵循从问题到解决方案再到证据的连贯路径？

### 检查

| 检查项 | 需核实的内容 | 发现严重程度 |
|-------|---------------|-----------------|
| 观察 → 差距推导 | 差距是否从观察中逻辑推导得出 | 重大 |
| 差距 → 洞见连接 | 关键洞见是否回应了已识别的差距 | 重大 |
| 洞见 → 解决方案对齐 | 解决方案架构是否实现了关键洞见 | 重大 |
| 解决方案 → 声明覆盖 | 声明是否涵盖解决方案的主要贡献 | 次要 |
| 跨层一致性 | 声明、树状结构和证据是否讲述同一个故事 | 重大 |
| 叙事完整性 | 激励性问题是否得到回答或被明确推迟 | 次要 |
| 差距覆盖 | 每个差距是否至少由一项声明在实质上加以回应 | 重大 |

### 评分锚点

| 分数 | 描述 |
|-------|-------------|
| 5 | 从观察→差距→洞见→解决方案→声明→证据有清晰的弧线，所有差距均得到回应 |
| 4 | 整体流畅，存在轻微缺口或一个未被回应的差距 |
| 3 | 整体流程存在，但层级间存在断裂 |
| 2 | 问题与声明之间存在显著不一致，或存在矛盾 |
| 1 | 无连贯的逻辑流程；各层讲述不同的故事 |

---

## D5. 探索完整性

**问题**：探索树是否如实记录了研究历程？

### 检查

| 检查项 | 需核实的内容 | 发现严重程度 |
|-------|---------------|-----------------|
| 死胡同具体性 | failure_mode 是否具体，lesson 是否可迁移 | 重大 |
| 决策理由质量 | 理由是否解释了为何选择的路径优于真实的替代方案 | 重大 |
| 已驳斥分支一致性 | 没有任何声明主张采用 dead_end 或 pivot 方法 | 关键 |
| 探索广度 | 主要设计选择是否有 ≥2 个已记录的替代方案 | 次要 |
| 诚实信号 | 树状结构是否记录了真实的负面结果，而非事后合理化 | 建议 |

### 评分锚点

| 分数 | 描述 |
|-------|-------------|
| 5 | 丰富的树状结构，具体的失败模式，可操作的经验教训，充分的理由说明，真实的负面结果 |
| 4 | 良好的树状结构，死胡同或决策文档存在轻微缺口 |
| 3 | 树状结构存在，但死胡同缺乏具体性或决策缺乏替代方案 |
| 2 | 模板化文档；死胡同和决策读起来流于形式 |
| 1 | 树状结构与声明矛盾，或完全读起来像事后合理化 |

---

## D6. 方法论严谨性

**问题**：实验设计是否合理，具备足够的基线和报告？

### 检查

| 检查项 | 需核实的内容 | 发现严重程度 |
|-------|---------------|-----------------|
| 基线充分性 | 比较对象是否正确？基线是否最新且相关？ | 重大 |
| 消融覆盖 | 多组件声明是否有隔离各个贡献的实验 | 重大 |
| 统计报告 | 是否提及方差、置信区间、运行次数或检验 | 定量声明为重大 |
| 指标-声明对齐 | 指标是否衡量了声明所断言的内容 | 重大 |
| 可复现性信号 | Setup 是否具体到足以复现（模型、数据集、硬件、超参数） | 次要 |

### 评分锚点

| 分数 | 描述 |
|-------|-------------|
| 5 | 全面的基线，适当的消融，统计严谨，指标与声明精确对齐 |
| 4 | 方法论扎实，存在轻微缺口 |
| 3 | 基本充分，但缺少部分基线或统计细节 |
| 2 | 存在显著缺口；比较性声明缺少基线，或没有消融实验 |
| 1 | 没有基线，没有消融，指标与声明不匹配 |

---

## 总体评级映射

| 评级 | 条件 |
|-------|-----------|
| **强烈接受** | 均值 ≥ 4.5 且无维度 < 3 |
| **接受** | 均值 ≥ 3.8 且无维度 < 2 |
| **弱接受** | 均值 ≥ 3.0 且无维度 < 2 |
| **弱拒绝** | 均值 ≥ 2.0 且（均值 < 3.0 或任一维度 < 2） |
| **拒绝** | 均值 < 2.0 或任一维度 = 1 |

## 发现严重程度定义

| 严重程度 | 含义 | 示例 |
|----------|---------|---------|
| `critical` | 根本性的认识论缺陷；声明或论证在当前形式下无法成立 | 因果声明仅由相关性支撑；声明主张采用死胡同方法 |
| `major` | 显著削弱声明或维度的重大缺陷 | 比较性声明无基线；平凡的可证伪标准；指标与声明不匹配 |
| `minor` | 不使工作无效的明显问题 | 缺少泛化边界；对冲语言与证据不一致 |
| `suggestion` | 建设性改进建议，而非缺陷 | 为上下文添加检索基线；记录探索广度 |
