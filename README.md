# Claude Code 学术写作技能库

基于 Claude Code 的学术科研技能集合，覆盖论文写作、润色、翻译、审稿、绘图等全流程。35 个技能均以中文命名，可在 Claude Code 中直接通过 `/技能名` 调用。

---

## 快速开始

### 1. 环境要求

- [Claude Code](https://claude.ai/code) CLI 已安装并登录
- Mac / Windows（Git Bash）均支持

### 2. 注册技能（首次使用）

技能文件需注册到 Claude Code 的全局命令目录才可调用。运行以下脚本自动完成注册：

```bash
# 方式一：运行注册脚本（自动检测路径，Mac/Windows 通用）
bash /path/to/技能/link_skills.sh

# 方式二：手动注册单个技能到全局命令目录
ln -sf "/path/to/技能/技能名/SKILL.md" ~/.claude/commands/技能名.md
```

> `/path/to/技能` 替换为实际克隆路径，例如 Mac 上为 `~/Desktop/技能`，Windows 上为 `/c/Users/<用户名>/Desktop/技能`。

> 注意：注册后需重启 Claude Code 才能加载新技能。

### 3. 调用技能

在 Claude Code 对话框中直接输入斜杠命令：

```
/中文润色
/英文润色
/机器学习论文写作
```

输入技能名后回车，技能会自动加载并提示你输入内容。

## 典型使用流程

### 场景一：从零开始写一篇 ML 论文

```
1. /研究灵感头脑风暴    → 发展研究想法
2. /科研创意思维        → 细化为完整研究方案
   ↓ 做实验，整理结果
3. /机器学习论文写作    → 提供代码仓库或实验结果，生成完整论文草稿
4. /实验分析           → 将实验数据转化为 LaTeX 分析段落
5. /论文配图设计        → 生成方法架构图
6. /图题生成 / /表题生成 → 规范图表标题
7. /英文润色           → 深度润色各节文本
8. /去AI化英文         → 消除 AI 痕迹
9. /逻辑核查           → 终稿红线检查
10. /审稿人模拟        → 模拟审稿，提前发现问题
```

### 场景二：中文草稿投英文顶会

```
1. /中文润色           → 先润色中文草稿
2. /中转英LaTeX        → 翻译为投稿级英文 LaTeX
3. /英文润色           → 深度润色英文
4. /去AI化英文         → 消除 AI 套话
5. /逻辑核查           → 终稿检查
```

### 场景三：论文投稿前自审

```
1. /审稿人模拟         → 模拟审稿人视角，获取 Weaknesses 和评分
2. 根据意见修改论文
3. /逻辑核查           → 确认修改后无逻辑矛盾
4. /英文润色           → 润色修改段落
```

### 场景四：快速理解一篇英文论文

```
1. /英转中LaTeX        → 粘贴关键章节 LaTeX，获取流畅中文翻译
```

## 目录结构

```
技能/
├── README.md                    # 本文档
├── link_skills.sh               # 技能注册脚本
├── 技能名/                      # 每个技能的独立目录
│   ├── SKILL.md                 # 技能定义（角色/任务/约束条件/输出格式）
│   ├── references/              # 参考文档（上下文加载）
│   │   └── *.md                 # 风格指南、检查清单、会议规则等
│   └── templates/               # LaTeX 模板（部分技能）
├── Output/                      # 技能输出文件存放目录
└── 参考论文/                    # 实验室参考论文原文
```

### 技能文件格式（SKILL.md）

每个 `SKILL.md` 遵循统一的 frontmatter 格式：

```yaml
---
name: 技能名称
description: 一句话描述
version: 1.0.0
author: personal
---

# 技能标题
（技能正文：角色 / 任务 / 约束条件 / 输出格式，章节标题统一使用中文）
```

## 添加新技能

### 方法一：手动创建

1. 在 `技能/` 下新建目录，如 `技能/新技能名/`
2. 创建 `SKILL.md`，参考现有技能格式填写
3. 注册到全局命令目录：
   ```bash
   # 方式一：使用注册脚本（推荐，自动处理所有技能）
   bash /path/to/技能/link_skills.sh

   # 方式二：手动创建符号链接
   ln -sf "/path/to/技能/新技能名/SKILL.md" ~/.claude/commands/新技能名.md
   ```
4. 重启 Claude Code

## 为技能添加实验室参考资料

将论文或风格指南放入技能的 `references/` 目录，并在 `SKILL.md` 的工作流部分注明加载步骤：

```
技能名/
  references/
    lab-style-guide.md     # 写作风格指南
    lab-paper-xxx.md       # 实验室代表作（PDF 提取文本）
```

在 `SKILL.md` 工作流中加入：

```
- [ ] Step N: 读取 references/lab-style-guide.md，参照实验室写作风格进行写作
```

> 技能库中的 `/机器学习论文写作` 已内置实验室风格指南（基于 TimePMG、TimeMRA、MSH-LLM 三篇代表作提炼），可直接参考其 `references/lab-style-guide.md`。

## 常见问题

**Q：输入 `/技能名` 后下拉列表没有显示？**
A：Claude Code 的命令选择器对中文输入匹配支持有限。直接输入完整技能名回车即可触发，无需依赖下拉补全。

**Q：提示 Unknown skill？**
A：技能未注册或注册后未重启 Claude Code。参考「注册技能」章节重新注册，然后重启。

**Q：如何更新技能内容？**
A：直接编辑对应的 `技能名/SKILL.md` 文件即可，无需重新注册。`link_skills.sh` 创建的是指向 `SKILL.md` 的符号链接（symlink），文件内容修改会实时生效。

**Q：注册命令的中文名是从哪里来的？**
A：`link_skills.sh` 会把符号链接命名为 `~/.claude/commands/技能目录名.md`，并要求对应 `SKILL.md` frontmatter 中的 `name:` 字段与目录名一致（均为中文），这样命令标识符和调用名称才能保持一致。
